1. WHAT THIS PACKAGE DOES
============================================================

Vitro Genesis is a Mudlet package for Genesis MUD. It adds several quality-of-life systems:

- AutoHunter with room-range hunting, target scanning, delays, profile support, pause/resume, and session tracking.
- Regular Mudlet Mapper follow/recovery support.
- GMCP-based mapper anchoring and repair tools.
- Genesis Magic Map display.
- Vitals HUD.
- Inventory/equipment HUD with blacklist filtering.
- Session Tracker HUD.
- Loot automation after kills.
- Path-mode hunting for areas with special exits.
- Manual scout mode for repairing GMCP room links.
- Ship/disembark automation and highlighting.
- Herbing helper.
- Kill-all alias that scans the room and attacks multiple visible mobs.
- Combat and utility highlights.
- In-client package help through phelp.

The package uses Genesis GMCP data when available, especially for room information and mapper recovery. It also uses normal Mudlet scripts, aliases, and triggers.


2. INSTALLING THE PACKAGE
============================================================

Recommended fresh install:

1. Open Mudlet.
2. Open or create your Genesis profile.
3. Connect to Genesis.
4. In Mudlet, open Package Manager.
5. Install/import the .mpackage file:
   Vitro Genesis V1.3.mpackage
6. Reconnect to Genesis or restart Mudlet.
7. After login, wait for the startup restore message.
8. Type:
   phelp setup

If the UI does not restore automatically after login, type:

   gstartup restore

Then check package status:

   gstartup status
   phelp
   gvitals status
   ginv status
   ghunt status
   gmap status


3. FIRST-TIME SETUP CHECKLIST
============================================================

After installing, do these in order:

1. Confirm the package loaded:
   phelp

2. Restore package windows:
   gstartup restore

3. Show the main HUDs:
   gvitals show
   ginv show
   gmagic show

4. Move HUDs where you want them.
   For draggable HUDs, use their drag commands or click/drag if supported:
   ginv grab
   ginv drop

5. Save layout:
   glayout save
   gmagic save
   ginv save

6. Check mapper state:
   gmap status
   gfollowfix audit

7. If you are in a mapped area but the regular Mudlet Mapper is on the wrong room, fix it:
   greanchor room <correct Mudlet room ID>
   gfollowfix audit

8. Clear scout range before repairing a new area:
   gscout clearrange
   gscout start

9. Walk around manually and repair GMCP links:
   look
   gscout fix

10. Stop scout mode when finished:
   gscout stop
   gmap save


4. RECOMMENDED SETUP ORDER
============================================================

Use this order when setting up a new profile:

A. Basic restore:
   gstartup restore
   glayout load
   gmagic show
   gvitals show
   ginv show

B. Save your window positions:
   glayout save
   gmagic save
   ginv save

C. Mapper verification:
   gmap status
   gfollowfix audit

D. Scout/repair current mapped area:
   gscout clearrange
   gscout clear
   gscout start

E. Walk through the area:
   look
   gscout fix

F. Turn normal systems back on:
   gscout stop
   gfollowfix on
   gmap save

G. Configure hunting:
   ghunt targets <target names>
   ghunt range <first room> <last room>
   ghunt startroom <room>
   ghunt save <profile>

H. Test hunting briefly:
   ghunt load <profile>
   ghunt start
   ghunt stop

============================================================

The Startup system restores the package UI after login.

Main commands:

   gstartup restore
      Manually restores package UI systems.

   gstartup status
      Shows startup status and whether restore is enabled.

   gstartup on
      Enables automatic startup restore.

   gstartup off
      Disables automatic startup restore.

   gstartup maptest
      Tests opening/restoring the regular Mudlet Mapper.

The Layout system saves and restores selected HUD/window positions.

Main commands:

   glayout save
      Saves regular Mapper, AutoHunter HUD, and SessionTracker HUD positions.

   glayout load
      Loads saved layout.

   glayout status
      Shows saved positions.

   glayout reset
      Clears saved layout data.

Separate saves:
   gmagic save
      Saves Genesis Magic Map position.

   ginv save
      Saves Inventory HUD position/size.

Suggested after moving windows:

   glayout save
   gmagic save
   ginv save

8. INVENTORY HUD
============================================================

The Inventory HUD shows:

   ⚔ Wielding ⚔
   🛡 Wearing 🛡
   🎒 Carrying 🎒

It intentionally ignores money/coins. It also filters out common autoload/weightless items.

Current built-in blacklist includes:

   tutorial journal
   magic map
   glowing magic map
   black signature book
   pair of combat boots
   combat boots
   newbie pin

Commands:

   ginv show
      Shows the Inventory HUD.

   ginv hide
      Hides the Inventory HUD.

   ginv toggle
      Toggles visibility.

   ginv refresh
      Sends inventory command and updates the HUD.

   ginv save
      Saves position/size.

   ginv load
      Loads position/size.

   ginv status
      Shows Inventory HUD status and blacklist count.

   ginv grab
      Starts mouse drag if supported.

   ginv drop
      Stops mouse drag.

   ginv move <x> <y>
      Manually moves the HUD.

   ginv size <width> <height>
      Resizes the HUD.

   ginv reset
      Resets position/size.

Auto-refresh:
The package includes an Inventory Auto Refresh alias that can refresh the HUD after inventory-changing commands such as get, take, drop, put, wield, wear, remove, open, close, fillpack, and similar.

If inventory becomes spammy:
- Make sure the Inventory Capture trigger is installed and active.
- Make sure the script uses send("i", false) during refresh.
- Increase auto-refresh delay if needed.

Genesis Magic Map is separate from the regular Mudlet Mapper. It displays mud-side map data and is often more reliable for current location than the regular mapper.

Commands:

   gmagic show

   gmagic lock
   gmagic unlock

   gmagic help

Recommended setup:

   gmagic show
   gmagic unlock
   move it where you want
   gmagic lock
   gmagic save

Important:
If the regular Mudlet Mapper disagrees with the Magic Map, trust the Magic Map first. The Magic Map is based on mud-side information. The regular mapper can be wrong if GMCP anchors are stale or missing.


11. REGULAR MUDLET MAPPER
============================================================

The regular Mudlet Mapper is Mudlet's normal map window. AutoHunter range mode uses regular Mudlet room IDs.

Important concepts:

Mudlet room ID:
   A number assigned by Mudlet, such as 602 or 11545.

Genesis GMCP room ID:
   A mud-side text ID, such as zxO820 or FAt44.

Room short:
   The room title from GMCP, such as "A pier at the port of Gelan".

The regular mapper must know which Mudlet room corresponds to the current GMCP room.

Commands:

   gmap status
      Shows mapper state, last Mudlet room ID, last Genesis room ID, exits, and current GMCP Room.Info.

   gmap follow
      Enables mapper following.

   gmap unfollow
      Disables mapper following.

   gmap center
      Centers map on current room.

   gmap save
      Saves Mudlet map data.

   gmap stop
      Stops creating rooms.

   gmap start
      Starts fresh mapping. Do not use this just to repair an anchor unless you intentionally want mapping mode.

   gmap wipe map
      Dangerous. Only use if you intentionally want to wipe map data.

Check current room:

   gmap status

Also useful:

   lua echo("\nMudlet player room: " .. tostring(getPlayerRoom()) .. "\n")


12. MAPPER RECOVERY, FOLLOWFIX, AND GMCP ANCHORING
============================================================

FollowFix listens for GMCP Room.Info and tries to snap the regular Mudlet Mapper to the matching Mudlet room.

This works only if the current GMCP room ID is already saved on the correct Mudlet room.

Commands:

   gfollowfix on
   gfollowfix off
   gfollowfix status
   gfollowfix check
   gfollowfix reload
   gfollowfix audit

Most important command:

   gfollowfix audit

Audit shows whether the current GMCP room is linked to exactly one Mudlet room.

Good state:

   Matches: 1
   Room: 602 key=genesis_id

Bad states:

   Matches: 0
      Current GMCP room is not linked to any Mudlet room.

   Matches: 2 or more
      Duplicate anchors exist. The mapper may snap to the wrong room.

Manual anchor:

   greanchor room <room ID>

Example:

   greanchor room 602
   gfollowfix audit

Use greanchor when:
- The Magic Map is correct but regular Mudlet Mapper is wrong.
- You disembark from a ship and regular mapper remains in the old area.
- You use special exits that Mudlet does not follow correctly.
- FollowFix audit shows missing or duplicate anchors.

Safe repair flow:

   gfollowfix off
   greanchor room <correct room ID>
   gscout fix
   gfollowfix audit
   gfollowfix on


13. SCOUT MODE / GMCP ROOM-LINK REPAIR
============================================================

Scout mode is used to repair or prime GMCP links on already mapped areas.

It links:

   current mud-side GMCP room ID -> current Mudlet room ID

Commands:

   gscout start
      Starts scout mode.

   gscout stop
      Stops scout mode.

   gscout status
      Shows scout stats.

   gscout audit
      Audits current room and current GMCP info.

   gscout fix
      Links current GMCP room to current Mudlet room.

   gscout clear
      Clears scout counters.

   gscout range <first> <last>
      Sets scout range.

   gscout clearrange
   gscout norange
      Clears scout range and allows any current Mudlet room.

   gscout fromhunt
      Manually pulls a range from ghunt/groute if available.

Important v1.3 behavior:
gscout start no longer automatically pulls the old ghunt range. This prevents old test ranges from blocking unrelated areas.

Recommended repair workflow:

   gfollowfix off
   gscout clearrange
   gscout clear
   gscout start

Then walk manually. In each room:

   look

If the regular Mudlet Mapper is on the correct room:

   gscout fix

If the regular Mudlet Mapper is on the wrong room:

   greanchor room <correct Mudlet room ID>
   gscout fix

When finished:

   gscout stop
   gfollowfix on
   gfollowfix audit
   gmap save

Warning:
Do not blindly walk fast through a messy area while scout is linking. Scout saves the current GMCP ID to whatever Mudlet room the mapper currently thinks you are in. If the mapper marker is wrong, anchor first.

15. AUTOHUNTER / GHUNT
============================================================

AutoHunter automates scanning, target selection, movement, attacks, looting support, and run/session logic.

Core commands:

   ghunt status
   ghunt start
   ghunt stop
   ghunt pause
   ghunt resume
   ghunt reset

Target setup:

   ghunt targets bunny, rabbit
   ghunt targets orc, goblin

Room range setup:

   ghunt range <first room> <last room>

Example:

   ghunt range 11516 11559

Start/home room:

   ghunt startroom <room ID>

Example:

   ghunt startroom 11511

Looping:

   ghunt loop on
   ghunt loop off

Delays:

   ghunt delay scan <seconds>
   ghunt delay attack <seconds>
   ghunt delay move <seconds>
   ghunt delay return <seconds>

Example:

   ghunt delay scan 1
   ghunt delay attack 0.7
   ghunt delay move 4

Stats:

   ghunt kills
   ghunt resetkills
   ghunt resetruns
   ghunt timer

Basic range-mode setup example:

   ghunt targets bunny
   ghunt range 11516 11559
   ghunt startroom 11511
   ghunt loop on
   ghunt delay scan 1
   ghunt delay attack 0.7
   ghunt delay move 4
   ghunt save bunnies

Start hunting:

   ghunt load bunnies
   ghunt status
   ghunt start

Stop hunting:

   ghunt stop

Important:
Range mode uses regular Mudlet Mapper room IDs, not GMCP IDs and not Magic Map coordinates. If the regular mapper is wrong, AutoHunter can be wrong. Prime/repair the area with gscout before long hunts.

17. PATH-MODE HUNTING / GPATH
============================================================

Path mode moves using a recorded list of commands instead of relying on regular mapper routing.

Use path mode for:
- swim east / swim west
- climb tree
- enter portal
- board/disembark situations
- areas where normal mapper following is unreliable
- special exits not represented as normal Mudlet exits

Commands:

   gpath record <name>
      Starts recording a path.

   gpath record stop
      Stops and saves recording.

   gpath record cancel
      Cancels recording.

   gpath list
      Lists saved paths.

   gpath show <name>
      Shows path steps.

   gpath use <name>
      Activates a path.

   gpath off
      Disables path mode.

   gpath pop
      Removes last recorded step.

   gpath clear
      Clears active path steps.

   gpath delete <name>
   gpath del <name>
      Deletes a saved path.

Recording example:

   gpath record riverbunnies
   swim east
   ne
   sw
   swim west
   gpath record stop

Use path:

   gpath use riverbunnies
   ghunt start

Path mode still lets AutoHunter scan, attack, kill, and loot. Only movement changes.

19. KILL-ALL ALIAS / KA
============================================================

The ka alias scans the room and attacks all visible targets matching the requested word.

Command:

   ka <target>

Example:

   ka orc

Behavior:
1. Sends a silent look.
2. Captures room text.
3. Counts matching mobs.
4. Sends kill commands for all visible matches.
5. Sends cdonk after the first kill command.

Example room line:

   Three orcish warriors.

Command:

   ka orc

Expected output commands:

   kill orc
   cdonk
   kill second orc
   kill third orc

The matcher supports prefix-style matching:

   orc -> orc, orcs, orcish
   rat -> rat, rats
   goblin -> goblin, goblins

Commands:

   ka <target>
   ka status

Important:
ka is a quick combat helper. It is not the same as full AutoHunter. It does not move through routes or manage hunting profiles.

   gherb command suranie

With optional herb override:

   gherb forest suranie

   gherb add river swim east, ne, sw, swim west
   gherb river suranie

Recording:

   gherb record <name>
   gherb step <movement command>
   gherb record stop
   gherb record cancel

Example:

   gherb record river
   gherb step swim east
   gherb step ne
   gherb step sw
   gherb step swim west
   gherb record stop
   gherb river

Herbing notes:
- It does not use GMCP.
- It reacts to Genesis herbing/search text.
- Use attentively and follow Genesis rules.

25. COMMON TROUBLESHOOTING
============================================================

Problem: phelp does not work.
Check:
   - Package imported correctly.
   - Package Help script exists and is active.
   - Package Help alias exists and is active.

Problem: UI did not restore after login.
Run:
   gstartup restore
   gstartup status

Problem: Login restore fires when seeing "Genesis ShipLines".
The startup trigger must be strict. It should match a real login line such as:
   ^Welcome to Genesis(?:\s.*)?$

Do not use a broad trigger like:
   Genesis

Problem: Magic Map is correct, but regular Mudlet Mapper is wrong.
Run:
   gmap status
   gfollowfix audit

Then repair:
   greanchor room <correct room ID>
   gscout fix
   gfollowfix audit

Problem: gscout says room is outside an old range.
Clear the range:
   gscout clearrange
   gscout start

Problem: gscout still has old warning counters.
Clear counters:
   gscout clear
   gscout status

Problem: AutoHunter moves wrong.
Likely mapper anchor issue.
Check:
   gmap status
   gfollowfix audit

Repair area:
   gfollowfix off
   gscout clearrange
   gscout start
   look
   greanchor room <correct room ID>
   gscout fix
   gscout stop
   gfollowfix on

Problem: Inventory HUD shows too many autoload items.
The current blacklist hides:
   tutorial journal
   magic map
   glowing magic map
   black signature book
   pair of combat boots
   combat boots
   newbie pin

If one still appears, it may have a slightly different exact name. Add that exact normalized name to the blacklist in the GenesisInventory script.

Problem: Inventory command is spammy.
Make sure Inventory refresh sends:
   send("i", false)

Make sure Inventory Capture trigger gags inventory lines while capturing.

Problem: ka orc finds no orcs when line says "Three orcish warriors."
Make sure GenesisKillAll version is:
   1.2.0-room-scan-prefix-match

Problem: ka sends too few or too many kill commands.
